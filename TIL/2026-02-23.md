## HTTP 프로토콜 특징

- **Connectionless**: 하나의 요청에 하나의 응답을 한 후 연결 종료. 새 요청이 있을 때마다 새로운 연결
- **Stateless**: 통신이 끝난 후 상태 정보 저장x
  -> 상태 유지 위해 **Cookie** 등장

## Cookie

- Key & Value 로 구성
- Server가 Client에게 발급
- Client가 Server에 요청을 보낼 때마다 쿠키를 같이 전송. Server는 쿠키를 보고 Client 구분

## Session

- Stateless인 웹 환경에서 사용자를 식별하기 위해 **서버 측에 데이터를 저장**하고, 클라이언트에게는 암호화된 'Session ID'만 쿠키로 쥐여주는 상태 유지 기술

## 쿠키와 세션의 차이

- 저장위치: 브라우저 / 서버
- 보안: 취약 / 우수
- 속도: 빠름 / 느림
- 서버 부하: 없음 / 있음
- 데이터 형태: 텍스트 정보 등 / 의미 없는 해시값(Session ID)
